Tutorial de Uso do GitHub

Passo 01: Criar Repositório no GitHub

1.1: Criar um repositório público ou privado no GitHub.

1.2: Adicionar uma descrição ao repositório.

1.3: Copiar o código HTTP ou SSH do repositório.

Passo 02: Abrir o Terminal e Navegar até o Diretório do Projeto

2.1: Abra o terminal.

2.2: Navegue até o diretório do seu projeto usando o comando:

cd\
cd C:\Users\igor_\Documents...

Passo 03: Abrir o VSCode

3.1: Abra o diretório do projeto no VSCode:

code .

3.2: Abra o terminal dentro do VSCode e insira os comandos.

Passo 04: No Terminal no VSCode (Branch Recém Criada)

4.1: Iniciar o repositório local:

git init

4.2: Vincular o repositório local ao repositório remoto do GitHub:

git remote add origin https://github.com/seu-usuario/nome-do-repositorio.git

4.3: Nomear a branch atual como "main":

git branch -M main

4.4: Preparar todos os arquivos do projeto para o repositório local:

git add .

4.5: Verificar o status do repositório local:

git status

4.6: Caso o Git ainda não esteja vinculado ao seu nome e email, configure-os com os seguintes comandos:

git config --global user.email "you@example.com"
git config --global user.name "Your Name"

4.7: Caso queira remover o repositório Git local e recomeçar, execute o comando opcional abaixo:

rm -rf .git

4.8: Confirmar as preparações feitas pelo git add e atribuir uma mensagem:

git commit -m "Atualização do código"

4.9: Subir as alterações para o repositório remoto (branch main):

git push origin main

Passo 05: Para Uma Branch Já Existente

5.1: Clonar o repositório remoto:

git clone url

5.2: Preparar todos os arquivos do projeto para o repositório local:

git add .

5.3: Confirmar as preparações feitas pelo git add e atribuir uma mensagem:

git commit -m "Atualização do código"

5.4: Subir as alterações para o repositório remoto (branch main):

git push origin main

Passo 06: Criar um Fork de um Repositório

6.1: Caso queira fazer um Fork de um repositório público:

Vá até o repositório desejado no GitHub e clique em "Fork" no canto superior direito. Isso cria uma cópia do repositório para sua conta. Agora você pode clonar esse repositório para o seu ambiente local.

6.2: No terminal do VSCode, entre no diretório desejado:

cd NomeDoProjeto

6.3: Clone o repositório Forked usando o comando abaixo:

git clone https://github.com/seu-usuario/nome-do-repositorio.git

Agora você tem uma cópia local do repositório para fazer alterações, como se fosse seu repositório.

Passo 07: Contribuir com um Projeto de Terceiros via Pull Request

7.1: Caso queira contribuir para um projeto de terceiros, siga os passos abaixo:

7.2: Após realizar as alterações desejadas em sua cópia local do repositório, envie as alterações para seu repositório no GitHub (Fork). No repositório de origem, clique em "Pull Request" para submeter suas alterações ao mantenedor do repositório original.

7.3: O mantenedor do repositório original analisará suas mudanças e, se aprovado, fará o merge com o repositório original.

7.4: No terminal, após fazer as alterações, execute os seguintes comandos:

git add .
git commit -m "Mensagem do seu Pull Request"
git push origin main

Agora, abra um Pull Request no GitHub para enviar suas alterações ao projeto original.
